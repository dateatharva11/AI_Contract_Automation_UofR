Functional Requirements 

FR-1: User Roles and Permissions  

Admin – Create, edit, submit and manage contracts and vendor information, and View dashboard 
Reviewer – View, edit and approve contracts 

Vendor – View and sign the final contract 

 

FR-2: Contract Intake Form 

The admin shall be able to create a new contract by providing: 

Project Name 

Project Number 

Vendor 

Project Dates 

Budget Amount 

The system must validate the required fields before proceeding. 

 

FR-3: AI-Based Contract Generation with RAG pipeline 

The system will embed historical contracts and templates, associating vectors to every data point using an AI API.  

These embeddings will be stored alongside contract and vendor data in a database. 

When the user provides details on the contract intake form, the input shall be embedded using the same AI embedding API. 

The system will compare the embedding of the input in the database using similarly search and top N relevant results (contracts/templates) will be retrieved. 

These results would be combined with the system prompt, and a tailored draft contract would be generated. 

The admin shall click on the “Generate Contract” option to run this flow and receive the AI-Generated Contract draft in an editable format.  

 

FR-4: Rich-Text Editor & Export Capability 

The system shall provide text editing and formatting options for the contract draft will version history logging. 

The user should be able to export the contract as PDF/DOCX file 

 

FR-5: Review and Approval Workflow 

Admin can submit the draft for “Review” and assign reviewer(s). 

Reviewers can view, edit, and add comments to the contract draft. It can also transition contract to the Vendor. 

Vendor can “Sign” the approved contract without the option to edit it. 

 

FR-6: Vendor Portal 

Vendor shall: 

View approved contracts 

Be restricted from editing it 

 

FR-7: Admin Dashboard 

Admin dashboard shall display: 

Active contracts 

Contracts pending review 

Recent Activity with the latest contracts updated first 

Contract pipeline bar graph with volume of contracts by stage (drafts, in review, approved and signed) 

 

FR-8: Vendor Directory 

Admin can have a vendor directory page that allows adding, editing, and removing vendors with contact information and more details about it.  

 

FR-9: Contract Directory 

Admin can view all contracts with details such as the status, vendor information and dates (project timeline). 

 

Non-functional Requirements 

Security 

Role-based access control 

Encrypted data at rest and in transit 

Audit logs must be immutable 

Performance 

Contract generation under 15 seconds 

Dashboard loading under 5 seconds 

Scalability 

Supporting increased contract database size 

Availability 

 95% uptime target 

Data Integrity 

Version history must be maintained 

Avoid data overwriting with parallel editing. 

AI Governance 

AI output must be reviewable 

 

Use Cases 

Use Case 1: Generate Contract 

Actor – Admin  

Precondition – Admin must be logged into the system 

Flow –  

Admin fills contract intake form 

Admin clicks “Generate Contract” 

System embeds input 

System performs vector search 

System generates draft 

Draft displayed in editor 

Postcondition – Admin must have an editable draft contract created by the system 

 

Use Case 2: Submit for Review 

Actor – Admin 

Precondition – Admin must contain a draft contract 

Flow –  

Admin clicks “Submit for Review” 

Reviewer notified 

Status updated to “In Review” 

Postcondition – Reviewer must have an editable draft contract passed by the admin 

 

Use Case 3: Review and Approve 

Actor – Reviewer 

Precondition – Reviewer must be logged into the system and should have a draft contract 

Flow –  

Reviewer edits document  

Reviewer clicks “Approve Contract” 

Status updated to “Approved” 

Postcondition – Vendor must have an editable draft contract passed by the reviewer 

 

Use Case 4: Vendor Signing 

Actor – Vendor  

Precondition – Vendor must be logged into the system 

Flow –  

Vendor views contract  

Vendor clicks “Sign”  

Postcondition – Contract status updated to “Executed” 

 

Use Case 5: Manage Vendors 

Actor – Admin  

Precondition – Admin must be logged into the system 

Flow –  

Admin navigates to vendor directory  

Adds or edits vendor  

Saves changes
